github 为什么要把密码换成 token ​

git 提交 报错如下：

shell
remote: Support for password authentication was removed on August 13, 2021.
1

github 为什么要把密码换成 token ​

官方解释

如何生成自己的 token ​

个人设置页面 => Setting => Developer settings => Personal access tokens => Generate new token

设置 ​

shell
git remote set-url origin https://<your_token>@github.com/<USERNAME>/<REPO>.git
1
  • <your_token>：换成你自己得到的 token
  • <USERNAME>：是你自己 github 的用户名
  • <REPO>：是你的仓库名称

5. 分支 ​

sh

7. 查看信息 ​

sh
# 显示变更的文件
git status

# 显示当前分支的版本历史
git log

# 显示commit历史，以及每次commit发生变更的文件
git log --stat

# 搜索提交历史，根据关键词
git log -S [keyword]

# 显示某个文件的版本历史，包括文件改名
git log --follow [文件路径]
git whatchanged [文件路径]

# 显示指定文件相关的每一次diff
git log -p [文件路径]

# 显示过去5次提交
git log -5 --pretty --oneline

# 显示所有提交过的用户，按提交次数排序
git shortlog -sn

# 显示指定文件是什么人在什么时间修改过
git blame [file]

# 显示暂存区和工作区的差异
git diff

# 显示工作区与当前分支最新 commit 之间的差异
git diff HEAD

# 显示今天你写了多少行代码
git diff --shortstat "@{0 day ago}"

# 显示当前分支的最近几次提交
git reflog

8. 提交 ​

sh
# 下载远程仓库的所有变动

# 上传本地指定分支到远程仓库
git push [remote] [branch]

# 强行推送当前分支到远程仓库，即使有冲突
git push [remote] --force

# 推送所有分支到远程仓库
git push [remote] --all

9. 撤销 ​

sh
# 重置暂存区的指定文件，与上一次 commit 保持一致，但工作区不变
git reset [文件路径]

# 重置暂存区与工作区，与上一次 commit 保持一致
git reset --hard

# 重置当前分支的指针为指定 commit，同时重置暂存区，但工作区不变
git reset [commit id]

# 重置当前分支的 HEAD 为指定 commit，同时重置暂存区和工作区，与指定 commit 一致
git reset --hard [commit id]

# 重置当前 HEAD 为指定 commit，但保持暂存区和工作区不变
git reset --keep [commit id]

# 新建一个 commit，用来撤销指定 commit 后者的所有变化都将被前者抵消，并且应用到当前分支
git revert [commit id]

# 暂时将未提交的变化移除，稍后再移入
git stash
git stash pop

将代码提交到 github 的 gh-pages 分支 ​

  1. 安装 gh-pages
sh
yarn add -D gh-pages
# OR npm install -D gh-pages
1
2
  1. 在 package.json 中添加如下脚本
json
"deploy": "gh-pages -d dist -m deploy",
"deploy:build": "npm run build && npm run deploy"
1
2
  1. 运行 deploy 脚本
sh
yarn deploy
# OR npm run deploy

删除 Git 中的所有提交历史记录 ​

提示 以 master 分支为例

sh
# 创建 orphan 分支
git checkout --orphan [分支名]

# 添加需要上传文件
git add .

# 提交更改
git commit -m "Initial"

# 删除需要清空提交记录的分支
git branch -D master

# 将当前分支重命名为需要清空提交记录的分支名
git branch -m master

# 强制更新存储库
git push -f origin master
